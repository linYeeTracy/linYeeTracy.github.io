# eslint

# 优点

  • 灵活：任何规则都可以开启闭合，以及有些规则有些额外配置
  • 很容易拓展和有需要可用插件
  • 容易理解产出
  • 包含了在其他检查器中不可用的规则，使得ESLint在错误检查上更有用
  • 支持ES6，唯一支持JSX的工具
  • 本身不支持ES7，但可以使用babel-eslint
  • 支持自定义报告

# 使用

  1. 先安装两个npm包
npm install eslint -g
npm install babel-eslint -g

>=4.7.0 to use eslint --fix.
>=4.14.0 to use with babel-eslint.

  1. 安装vscode eslint插件

  2. 根目录编写配置文件.eslintrc.js

参数说明：

(1) 环境(env)：设置你的脚本的目标运行环境，如browser，amd，es6，commonjs等，每种环境有预设的全局变量 (2) 继承(extends)：继承基础配置中的已启用的规则 (3) 规则(rules)：设定的规则及该规则对应的报错level (4) 解析器(parser)：配置解析器(Specifying Parser)，需要本地npm模块， Espree(默认) Esprima Babel-ESLint(支持ES7) (5) 语法(ecmaFeatures)：指定你想要支持的 JavaScript 语言选项，默认支持ECMAScript 5，可以设置启用对 ECMAScript 其它版本和 JSX 的支持 (6) 插件(plugins): 支持使用第三方插件，插件名称可以省略 eslint-plugin- 前缀

# 规则的严重性

"off" or 0 - turn the rule off 不验证 
"warn" or 1 - turn the rule on as a warning(doesn’ t affect exit code) 警告 "error" or 2 - turn the rule on as an error(exit code is 1 when triggered) 错误
（ 如有） 规则的选项(additional options)
"quotes": [2, "double"]
